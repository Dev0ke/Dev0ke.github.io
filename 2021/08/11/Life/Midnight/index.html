Part1

前面问的比较基础，基本都是《加密与解密》里的内容。时间过的比较久，顺序与当时的不太一样，可能会有部分遗漏。

  1. 断点类型有哪些？

    软件断点、硬件断点、条件断点、内存断点、消息断点。

  2. 软件断点的实现？

    修改断点所在位置的指令为0xCC，到达此处便会触发异常。内部实现可以参考4.软件断点。

  3. 内存断点的实现？

    内存断点的实现方式是将你欲下断地址所在的内存页增加一个名为PAGE_NOACCESS的属性，这个属性会把当前内存页设为禁止任何形式的访问，如果进行访问会触发一个内存访问异常。在这同时，调试器开始捕获目标程序中出现的这个异常，并判断触发这个异常的位置是否跟你下断的地址相同，如果相同则内存断点触发，暂停被调试程序的运行，否则放行。

  4. 硬件断点有了解吗？

    只答出到CPU中内置有4个调试寄存器。具体可以参考[原创]硬件断点和原理与实现

  5. 怎么设计无限硬件断点？

    比较少见的技术，[原创]Unlimited Break Works 无限硬件中断

  6. 函数调用约定有哪些，有什么特点？

    stdcall __cdecl fastcall thiscall x64
    栈平衡 子程序 调用者 子程序 参数个数不确定时调用者,否则子程序 调用者
    VARARG参数个数可变 否 是 否 是 是
    使用寄存器 否 否 是 this指针存入RCX 前4个参数RCX DX R8 R9（栈预留了4个参数的空间）
    注 Win32 API C规范 C++类传参 浮点参数使用4个XMM寄存器

    Visual C++ 编译器参数传递和命名约定

    常见函数调用约定(x86、x64、arm、arm64)

  7. 可变参数有了解过吗？

    没了解过，可参考从printf谈可变参数函数的实现

  8. 函数怎么实现返回值？

    放rax。
