Ollvm Deobfuscation Learning

简介

ollvm

Home · obfuscator-llvm/obfuscator Wiki (github.com)

Obfuscator-LLVM 是 2010 年 6 月由 Yverdon-les-Bains 的瑞士西部应用科学与艺术大学 (HEIG-VD) 的信息安全小组发起的一个项目。

该项目的目的是提供 LLVM 编译套件的开源分支，能够通过代码混淆和防篡改提供更高的软件安全性。

它包含三种混淆保护方式。

控制流平坦化Control Flow Flattening

​ CTF比赛中比较常见的混淆保护方式

混淆后的块分为

  • 序言Prologue

    函数开始地址对应的基本块

  • 主分发器Main dispatcher

    序言的后继为主分发器

  • 子分发器Sub dispatcher

    • 真实块Relevant blocks

      后继为预处理器的块为真实块

    • 返回块Retn

      无后继的块为retn块

  • 预处理器Pre dispatcher

    后继为主分发器的块为预处理器

虚假控制流Bogus Control Flow

向基本块中随机添加垃圾指令，同时会添加一些跳转语句和用于条件跳转的常量，会产生多个基本块。一种通用性较强的解决方法是

Hex-Rays: 十步杀一人，两步秒OLLVM-BCF

指令代换Instructions Substitution

将标准二元运算符（如加法、减法或布尔运算符）替换为功能等效但更复杂的指令序列。

  • Addition

    • a = b - (-c)
    • a = -(-b + (-c))
    • r = rand (); a = b + r; a = a + c; a = a - r
    • r = rand (); a = b - r; a = a + b; a = a + r

  • Subtraction

    • a = b + (-c)
    • r = rand (); a = b + r; a = a - c; a = a - r
    • r = rand (); a = b - r; a = a - c; a = a + r

  • AND

    • a = b & c => a = (b ^ ~c) & b

  • OR

    • a = b | c => a = (b & c) | (b ^ c)

  • XOR

    • a = a ^ b => a = (~a & b) | (a & ~b)

deflat源码解析

deflatten

Step1 获取序言、返回块、主分发器、预处理器

不需要符号执行，直接通过块与块之间的关联可以判断，

  • 入度为0为序言，出度为0为返回块
  • 序言的下一个块为主分发器
  • 除了序言之外，主分发器的前一块为预处理器

Step2 获取关联块、无用块

判断规则为：后继块为预处理器的块且指令数量大于8

Step3 符号执行前的预处理

为了方便，这里把真实块、序言和retn块统称为真实块，首先从每个真实块的起始地址开始进行静态分析

  1. 如果遇到CMOV指令，则代表该块后继存在分支，需要符号执行操作
  2. 遇到CALL指令，加入hook_addrs集合，在符号执行时HOOK并直接返回

Step4 符号执行

符号执行的主要操作是：如果遇到分支，就改变判断值执行两次来获取分支的地址，这里用angr的inspect在遇到类型为ITE的IR表达式时，改变临时变量的值来实现

控制流平坦化

debogus

基本思想是用angr找出所有可到达块，保留可到达块，NOP掉虚假块。

  1. 符号执行：创建一个可达块集合，通过angr将可达块加入该集合。

  2. 遍历所有块，如果不在可达块集合，则使用NOP填充。如果可达，则需要进一步判断其后继块的可达性。

    1. 如果后继块不可达，使用NOP填充不可达的后继块

    2. 如果后继块可达，将其加入跳转集合

    3. 如果最终跳转集合只有一个后继块，则将条件跳转指令PATCH为直接跳转JMP到该后继块。
